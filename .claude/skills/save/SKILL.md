# Design Highlights

Use this skill to summarize current design direction and keep a running project design memory.

## Source of Truth

- Read and update: .claude/design-highlights.md

## When To Use

- After meaningful UI or UX decisions.
- When architecture or workflow design changes.
- At the end of design-heavy sessions.
- When asked to summarize current design for future sessions.

## What To Capture

- Current design goals
- Visual style direction
- Key interaction patterns
- Important tradeoffs and constraints
- Open questions and pending decisions
- Next design steps

## Writing Guidelines

- Keep entries concise and scannable.
- Prefer bullets over long paragraphs.
- Record facts and decisions, not chatter.
- Include dates for each update.

## Update Procedure

1. Read the existing highlights file.
2. Preserve useful context.
3. Add a new dated update block.
4. Keep the top-level summary current.
